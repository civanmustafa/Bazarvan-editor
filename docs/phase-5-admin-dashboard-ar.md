# المرحلة الخامسة: لوحة الأدمن

تم تنفيذ لوحة الأدمن داخل شاشة لوحة التحكم نفسها.

تم تأجيل n8n بالكامل في هذه المرحلة.

## ماذا يرى الأدمن؟

إذا كان حسابك في جدول `profiles` يحمل:

```text
role = admin
```

فستظهر في أعلى لوحة التحكم منطقة جديدة باسم:

```text
جدول المستخدمين
```

هذه المنطقة تعرض:

- كل المستخدمين.
- دور كل مستخدم: `admin` أو `user`.
- عدد مقالات كل مستخدم.
- آخر حفظ.
- الوقت المستغرق.
- أهم الكلمات المفتاحية الرئيسية.

## كيف تستخدم جدول المستخدمين؟

1. سجل الدخول بحساب الأدمن.
2. افتح لوحة التحكم.
3. اضغط على أي مستخدم داخل جدول المستخدمين.
4. ستتغير قائمة المقالات أسفل الجدول لتعرض مقالات هذا المستخدم فقط.
5. اضغط `كل المستخدمين` للعودة إلى عرض كل المقالات.

## ماذا يظهر عند اختيار مستخدم؟

بعد اختيار مستخدم، سترى:

- مقالاته فقط.
- عدد المقالات.
- آخر حفظ.
- الوقت المستغرق.
- الكلمات المفتاحية داخل كل بطاقة مقال.
- نتائج التحليل المختصرة:
  - عدد الكلمات.
  - مخالفات الكلمات المفتاحية.
  - مخالفات الهيكل.
  - التكرارات.

## تفاصيل المقالة

أصبح في كل بطاقة مقال زر تفاصيل بأيقونة عين.

عند الضغط عليه تظهر نافذة تفاصيل كبيرة تحتوي على:

- عنوان المقالة.
- صاحب المقالة.
- الحالة.
- المصدر.
- نوع الظهور.
- لغة المقال.
- عدد مرات الحفظ.
- آخر حفظ.
- الوقت المستغرق.
- الكلمات المفتاحية:
  - الرئيسية.
  - الثانوية.
  - الشركة.
  - LSI.
- نتائج التحليل.
- بيانات الهدف `goal_context`.
- ملخص التحليل الكامل.
- معاينة النص.

ومن داخل نافذة التفاصيل يمكنك الضغط على:

```text
فتح في المحرر
```

لفتح المقالة وتعديلها.

## ماذا يرى المستخدم العادي؟

المستخدم العادي لا يرى جدول المستخدمين.

هو يرى مقالاته فقط حسب سياسات Supabase.

## إذا لم يظهر جدول المستخدمين

تأكد أن حسابك Admin عبر Supabase SQL Editor:

```sql
select id, email, role
from public.profiles
where email = 'YOUR_EMAIL@example.com';
```

إذا ظهر `role = user`، اجعله admin:

```sql
update public.profiles
set role = 'admin'
where email = 'YOUR_EMAIL@example.com';
```

بعدها:

1. سجل الخروج من المحرر.
2. سجل الدخول مرة أخرى.
3. افتح لوحة التحكم.

## إذا ظهر جدول المستخدمين لكن لا تظهر كل المقالات

تأكد أن المرحلة الأولى الخاصة بسياسات RLS مطبقة.

اختبر داخل Supabase:

```sql
select public.is_admin();
```

يجب أن ترجع `true` عندما تكون داخل جلسة الأدمن من التطبيق. إذا كنت تجرب من SQL Editor فقد تختلف الجلسة، لذلك الأهم أن يكون `role` في `profiles` هو `admin`.

## هل نحتاج شيئًا منك الآن؟

غالبًا لا.

فقط تأكد من:

1. حسابك في `profiles` هو `admin`.
2. لديك مقالات محفوظة في جدول `articles`.
3. سجلت خروجًا ودخولًا بعد تغيير الدور إلى admin.

إذا ظهرت رسالة خطأ في Console أو لم تظهر البيانات، أرسل لي نص الخطأ كما هو.
