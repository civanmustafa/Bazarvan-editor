# تحديث الحجز والأتمتة الذكية عند تعيين المستخدم

هذا التحديث يغير مكان تشغيل الأتمتة:

- لا يتم تشغيل توليد الصيغ البديلة أو LSI عند فتح المقالة.
- لا يتم تشغيل تحليل Gemini Pro عند فتح المقالة.
- يتم التشغيل فقط عند حجز المسودة بزر `احجز` أو عند تعيين مستخدم للمقالة من لوحة التحكم.
- التشغيل يتم من السيرفر عبر المسار:

```text
/api/articles/assigned-automation
```

## ماذا يحدث بعد الحجز أو تعيين المستخدم؟

1. السيرفر يقرأ المقالة مباشرة من Supabase.
2. إذا كانت المقالة `draft` وتحتوي على كلمة مفتاحية أساسية وسياق صفحة مكتمل، يشغل Gemini المجاني لتوليد:
   - الصيغ البديلة.
   - كلمات LSI.
   - عنوان مقترح فقط إذا كان العنوان فارغا أو عاما.
3. يتم دمج الكلمات الجديدة فوق الكلمات الموجودة دون حذف القديم.
4. بعدها يشغل Gemini Pro للأمر الجاهز `أفكار جديدة/متضاربة مع منافسين` فقط إذا توفرت كل الشروط:
   - نصوص المنافسين موجودة.
   - الكلمة المفتاحية الأساسية موجودة.
   - الصيغ البديلة موجودة.
   - كلمات LSI موجودة.
   - سياق الصفحة مكتمل: `pageType`, `objective`, `audienceScope`, `searchIntent`.
5. يتم حفظ نتيجة Gemini Pro داخل بيانات المقالة في Supabase حتى تظهر عند فتح المقالة لاحقا.

## متى لا يعمل التحليل؟

لن يعمل Gemini Pro تلقائيا إذا كان أحد الشروط ناقصا. مثلا:

- لا توجد نصوص منافسين.
- لا توجد كلمة مفتاحية أساسية.
- لا توجد صيغ بديلة أو LSI حتى بعد محاولة التوليد.
- سياق الصفحة ناقص.
- حالة المقالة ليست `draft`.
- مفاتيح Gemini Pro غير مضافة على السيرفر.

## مهم جدا

فتح المقالة لا يشغل أي أتمتة تلقائية الآن.
إذا أردت إعادة تشغيل الأتمتة، قم بإعادة تعيين المستخدم أو اجعل مستخدما يحجز المسودة المتاحة.

## متغيرات السيرفر المطلوبة على Hostinger

تأكد أن ملف البيئة الذي يستخدمه PM2 يحتوي على:

```bash
SUPABASE_URL=https://YOUR_PROJECT.supabase.co
SUPABASE_SERVICE_ROLE_KEY=YOUR_SERVICE_ROLE_KEY
GEMINI_API_KEYS=free_key_1,free_key_2
GEMINI_PAID_API_KEYS=paid_or_pro_key_1,paid_or_pro_key_2
GEMINI_MODEL=gemini-2.5-flash
GEMINI_PAID_MODEL=gemini-2.5-pro
ASSIGNED_ARTICLE_AI_PAUSE_MS=5000
```

لا تضع `SUPABASE_SERVICE_ROLE_KEY` أو مفاتيح Gemini المدفوعة داخل المتصفح أو داخل `VITE_`.

## خطوات تحديث Hostinger بعد وصول التعديل إلى GitHub

نفذ على السيرفر:

```bash
cd /var/www/bazarvan-editor
git pull origin main
set -a
source .env.server
set +a
npm run build
pm2 restart bazarvan-editor --update-env
pm2 save
```

بعدها جرّب:

1. افتح لوحة التحكم.
2. اختر مقالة `draft` غير مخصصة واضغط `احجز`، أو عيّن مستخدما من لوحة الأدمن.
3. انتظر قليلا لأن Gemini قد يستغرق وقتا. النظام ينتظر افتراضيا 5 ثوان بين توليد الصيغ و LSI وبين تحليل Gemini Pro.
4. افتح المقالة وتأكد أن الصيغ و LSI أضيفت.
5. إذا كانت نصوص المنافسين موجودة وبقية الشروط مكتملة، ستجد نتيجة Gemini Pro محفوظة في بيانات المقالة.
