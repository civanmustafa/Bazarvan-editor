# خطوات تحديث الحجز والأتمتة الذكية

هذا التحديث يضيف أو يعدل ما يلي:

- زر `احجز` للمقالات العامة غير المخصصة لأي مستخدم.
- عند حجز مسودة، يتم فتحها في المحرر تلقائيا حتى تعمل الأتمتة داخل سياق المقالة.
- توليد الصيغ البديلة و LSI تلقائيا باستخدام Gemini المجاني بعد الحجز، مع الإضافة على الموجود دون استبداله.
- تشغيل تحليل Gemini Pro للأمر الجاهز `أفكار جديدة/متضاربة مع منافسين` تلقائيا بعد الحجز فقط عند اكتمال كل الشروط.
- مسح نتائج وسجل الذكاء الاصطناعي عند تحويل حالة المقالة إلى `جاهز`.
- لا يتم حذف سجل الذكاء الاصطناعي عند إغلاق المقالة أو الانتقال بين المقالات؛ يتم حفظه لكل مقالة محليا ثم استعادته عند فتحها.
- بقاء القيمة التقنية للحالة داخل قاعدة البيانات `in_review`، مع عرضها للمستخدم باسم `جاهز`.

## 1. تحديث Supabase

افتح Supabase ثم SQL Editor وشغل الملفين التاليين بالترتيب إذا لم تكن شغلتهما سابقا:

```text
supabase/migrations/20260702020000_claim_available_articles.sql
supabase/migrations/20260702030000_clear_ai_results_on_ready_status.sql
```

الملف الأول ينشئ دالة:

```sql
public.claim_available_article(article_id)
```

وظيفتها:

- لا تعمل إلا لمستخدم مسجل دخول.
- لا تعمل إلا على مقالة `public` غير محجوزة.
- تنقل المقالة إلى المستخدم الذي ضغط `احجز`.
- تجعل `visibility = private`.
- تملأ `visibleToEmailsCsv` ببريد المستخدم.
- تمنع ظهور المقالة للمستخدمين الآخرين بعد تحديث اللوحة أو وصول Realtime.

الملف الثاني يحدث دالة تغيير الحالة بحيث تمسح `metadata.aiResults` عند تحويل المقالة إلى:

```text
in_review
```

وهي القيمة الداخلية التي تظهر في الواجهة باسم:

```text
جاهز
```

## 2. شروط التشغيل التلقائي بعد الحجز

توليد الصيغ البديلة و LSI يعمل بعد حجز المسودة إذا توفرت:

- حالة المقالة `draft`.
- الكلمة المفتاحية الأساسية.
- سياق الصفحة مكتمل: `pageType`, `objective`, `audienceScope`, `searchIntent`.

يتم دمج النتائج مع الكلمات الموجودة مسبقا، ولا يتم حذف أو استبدال الصيغ البديلة و LSI الحالية.

تحليل Gemini Pro للأمر الجاهز `أفكار جديدة/متضاربة مع منافسين` لا يعمل إلا عند توفر كل ما يلي:

- نص واحد على الأقل من نصوص المنافسين.
- الكلمة المفتاحية الأساسية.
- صيغ بديلة موجودة.
- كلمات LSI موجودة.
- سياق الصفحة مكتمل: `pageType`, `objective`, `audienceScope`, `searchIntent`.

إذا كان أي شرط ناقصا، لا يتم تشغيل تحليل Gemini Pro تلقائيا.

ملاحظة: إذا فشل Gemini بسبب مفتاح API أو خطأ مؤقت، لا يتم تسجيل العملية كأنها نجحت، ويمكن أن يعيد النظام المحاولة بعد إعادة فتح المقالة أو بعد توفر الشروط.

## 3. تحديث n8n

يمكنك إرسال الحالة بإحدى القيم التالية:

```json
{
  "status": "draft"
}
```

أو:

```json
{
  "status": "ready"
}
```

أو:

```json
{
  "status": "جاهز"
}
```

داخل قاعدة البيانات ستبقى القيمة التقنية:

```text
in_review
```

## 4. تحديث Hostinger

بعد رفع التعديل إلى GitHub، نفذ على السيرفر:

```bash
cd /path/to/editor
git pull origin main
npm ci
npm run build
pm2 restart bazarvan-editor --update-env
```

بعد ذلك افتح Supabase وشغل ملف SQL الجديد المذكور في الخطوة الأولى إذا لم يكن مطبقا.
