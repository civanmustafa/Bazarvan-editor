# خطوات تحديث الحجز والأتمتة الذكية

هذا التحديث يضيف:

- زر `احجز` للمقالات العامة غير المخصصة لأي مستخدم.
- تحويل المقالة بعد الحجز إلى مقالة خاصة بالمستخدم الذي حجزها.
- تشغيل توليد الصيغ البديلة و LSI تلقائيًا للمسودات عند توفر الكلمة الأساسية وسياق الصفحة.
- تشغيل تحليل Gemini المجاني للمنافسين تلقائيًا عند وجود نصوص منافسين محفوظة.
- مسح نتائج وسجل تحليلات الذكاء الاصطناعي عند إغلاق المقالة أو الانتقال لمقالة أخرى.
- تغيير تسمية حالة `in_review` إلى `جاهز`.

## 1. تحديث Supabase

افتح Supabase ثم SQL Editor وشغل الملف التالي:

```text
supabase/migrations/20260702020000_claim_available_articles.sql
```

هذا الملف ينشئ دالة:

```sql
public.claim_available_article(article_id)
```

وظيفتها:

- لا تعمل إلا لمستخدم مسجل دخول.
- لا تعمل إلا على مقالة `public` غير محجوزة.
- تنقل المقالة إلى المستخدم الذي ضغط `احجز`.
- تجعل `visibility = private`.
- تملأ `visibleToEmailsCsv` ببريد المستخدم.
- تمنع ظهور المقالة للمستخدمين الآخرين بعد تحديث اللوحة أو وصول تحديث Realtime.

## 2. تحديث n8n

يمكنك إرسال الحالة بإحدى القيم التالية:

```json
{
  "status": "draft"
}
```

أو:

```json
{
  "status": "ready"
}
```

أو:

```json
{
  "status": "جاهز"
}
```

داخل قاعدة البيانات ستبقى القيمة التقنية:

```text
in_review
```

لكن في المحرر ولوحة التحكم ستظهر للمستخدم باسم:

```text
جاهز
```

## 3. شروط التشغيل التلقائي للذكاء الاصطناعي

التشغيل التلقائي لا يبدأ إلا إذا كانت المقالة:

- حالتها `draft`.
- لديها كلمة مفتاحية أساسية.
- لديها سياق صفحة مكتمل: `pageType`, `objective`, `audienceScope`, `searchIntent`.

عند وجود صيغ بديلة أو كلمات LSI مسبقًا، سيضيف النظام الجديد فوقها بدون استبدالها.

## 4. تحديث Hostinger

بعد رفع التعديل إلى GitHub، نفذ على السيرفر:

```bash
cd /path/to/editor
git pull origin main
npm ci
npm run build
pm2 restart bazarvan-editor --update-env
```
