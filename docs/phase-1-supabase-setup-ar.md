# المرحلة الأولى: إعداد قاعدة بيانات Supabase

هذا الدليل يشرح لك كيف تطبق المرحلة الأولى بدون خبرة برمجية. أنا جهزت لك ملف SQL داخل المشروع، ودورك فقط أن تنسخه إلى لوحة Supabase وتشغله.

## ماذا تم تجهيزه؟

تم إنشاء الملف:

`supabase/migrations/20260701000000_phase_1_multi_user_schema.sql`

هذا الملف ينشئ:

- جدول `profiles`: بيانات المستخدمين وصلاحياتهم.
- جدول `articles`: كل المقالات في مكان مركزي بدل `localStorage`.
- جدول `article_versions`: نسخ اختيارية من المقالات عند الحاجة لاحقًا.
- جدول `n8n_ingest_logs`: سجل استقبال المقالات القادمة من n8n لاحقًا.
- صلاحيات `admin` و `user`.
- سياسات حماية RLS حتى لا يرى المستخدم مقالات غيره.
- صلاحية الأدمن لرؤية كل المستخدمين وكل المقالات.
- تجهيز أولي لمقالات n8n عبر حقول مثل `source`, `visibility`, `external_id`.

## قبل أن تبدأ

تحتاج فقط:

- حساب Supabase.
- مشروع Supabase جديد أو موجود.
- أن تعرف البريد الإلكتروني الذي تريد جعله حساب الأدمن.

مهم جدًا: لا ترسل لي ولا لأي شخص `service_role key`. هذا المفتاح سري جدًا ويجب أن يبقى داخل Supabase أو n8n أو السيرفر فقط.

## الخطوة 1: افتح مشروع Supabase

1. افتح موقع Supabase.
2. ادخل إلى حسابك.
3. افتح المشروع الذي تريد استخدامه للمحرر.
4. من القائمة الجانبية اختر `SQL Editor`.
5. اضغط `New query`.

## الخطوة 2: شغل ملف المرحلة الأولى

1. افتح الملف من جهازك:

   `supabase/migrations/20260701000000_phase_1_multi_user_schema.sql`

2. انسخ محتوى الملف بالكامل.
3. الصقه في صفحة `SQL Editor` داخل Supabase.
4. اضغط `Run`.
5. انتظر رسالة نجاح.

إذا ظهرت رسالة نجاح، فهذا يعني أن المرحلة الأولى من قاعدة البيانات تم إنشاؤها.

## الخطوة 3: أنشئ أول مستخدم

1. من قائمة Supabase الجانبية افتح `Authentication`.
2. اختر `Users`.
3. اضغط `Add user`.
4. اكتب بريدك الإلكتروني وكلمة مرور.
5. احفظ المستخدم.

بعد إنشاء المستخدم، سيقوم Supabase تلقائيًا بإنشاء سجل له داخل جدول `profiles`.

## الخطوة 4: اجعل حسابك Admin

بعد إنشاء المستخدم، ارجع إلى `SQL Editor` وشغل هذا الأمر بعد تعديل البريد:

```sql
update public.profiles
set role = 'admin'
where email = 'PUT_YOUR_EMAIL_HERE';
```

مثال:

```sql
update public.profiles
set role = 'admin'
where email = 'admin@example.com';
```

بعدها شغل هذا الأمر للتأكد:

```sql
select id, email, role, created_at
from public.profiles
order by created_at desc;
```

يجب أن ترى بريدك وبجانبه:

```text
admin
```

## الخطوة 5: تأكد أن الجداول موجودة

شغل هذا الأمر في `SQL Editor`:

```sql
select table_name
from information_schema.tables
where table_schema = 'public'
  and table_name in ('profiles', 'articles', 'article_versions', 'n8n_ingest_logs')
order by table_name;
```

يجب أن تظهر هذه الجداول:

```text
article_versions
articles
n8n_ingest_logs
profiles
```

## ماذا تعني الصلاحيات؟

لدينا نوعان من المستخدمين الآن:

- `admin`: يرى كل المستخدمين وكل المقالات.
- `user`: يرى مقالاته فقط، وأي مقالات مشتركة للجميع.

ولدينا أنواع ظهور للمقالات:

- `private`: تظهر لصاحب المقالة فقط.
- `shared`: تظهر لكل المستخدمين داخل المحرر.
- `team`: محفوظة للمستقبل إذا أضفنا فرق عمل.
- `public`: محفوظة للمستقبل إذا أردت عرض مقالات عامة.

ولدينا مصدر المقالة:

- `manual`: مقالة كتبها المستخدم داخل المحرر.
- `n8n`: مقالة وصلت من n8n.
- `import`: مقالة مستوردة بطريقة أخرى.
- `system`: مقالة أنشأها النظام.

## ماذا لو لم يظهر المستخدم في profiles؟

هذا نادر، لكن إذا أنشأت المستخدم ولم يظهر داخل `profiles`، شغل هذا الأمر:

```sql
insert into public.profiles (id, email, role)
select u.id, u.email, 'user'
from auth.users u
where not exists (
  select 1
  from public.profiles p
  where p.id = u.id
);
```

ثم كرر خطوة جعل حسابك `admin`.

## ماذا لو ظهر خطأ؟

إذا ظهر خطأ عند تشغيل SQL:

1. لا تغير السكربت بنفسك.
2. انسخ رسالة الخطأ كاملة.
3. أرسلها لي كما هي.

أمثلة على أخطاء محتملة:

- `permission denied`: غالبًا أنت داخل مشروع Supabase لا تملك صلاحية كاملة عليه.
- `relation auth.users does not exist`: غالبًا لم تشغل السكربت داخل Supabase SQL Editor.
- خطأ في البريد عند جعل الأدمن: تأكد أن البريد مطابق تمامًا للبريد الموجود في `Authentication > Users`.

## ما الذي أصبح جاهزًا الآن؟

بعد تطبيق هذا الملف في Supabase سيكون عندك الأساس المطلوب للمرحلة التالية:

- قاعدة بيانات مركزية.
- مستخدمون حقيقيون.
- أدمن ومستخدم عادي.
- حماية تمنع المستخدم من رؤية مقالات غيره.
- تجهيز لاستقبال مقالات n8n لاحقًا.

## ما الذي لم ننفذه بعد؟

هذه المرحلة لا تربط التطبيق بعد بـ Supabase. التطبيق ما زال يستخدم `localStorage` حتى ننفذ المرحلة الثانية والثالثة.

المراحل القادمة ستكون:

1. ربط تسجيل الدخول في المحرر بـ Supabase Auth.
2. تحويل حفظ المقالات من `localStorage` إلى جدول `articles`.
3. تعديل لوحة المستخدم ولوحة الأدمن.
4. إضافة endpoint آمن لـ n8n.

## المطلوب منك الآن

بعد أن تشغل السكربت:

1. أنشئ مستخدمًا في Supabase.
2. اجعل هذا المستخدم `admin`.
3. أرسل لي تأكيدًا أن الجداول ظهرت وأن حساب الأدمن أصبح `admin`.

إذا ظهر أي خطأ، أرسل لي نص الخطأ وسأعدله لك خطوة بخطوة.
